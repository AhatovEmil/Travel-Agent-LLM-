# Project Vision — AI Technical Founder

## Анализ идеи

**Что хочет пользователь?**
Платформу, в которой ИИ-агент выступает техническим сооснователем: принимает бизнес-идею
текстом и превращает её в готовый MVP-проект «под ключ» — с анализом, планом, архитектурой
и сгенерированным кодом, который можно скачать и запустить.

**Какая проблема решается?**
Основатели без технической команды не могут быстро проверить идею. Заказная разработка
дорога (тысячи $) и медленна (недели/месяцы). Продукт сокращает путь «идея → работающий
прототип» до минут и делает процесс прозрачным (видны все этапы и артефакты).

**Аудитория**
- Непрограммирующие фаундеры и предприниматели, проверяющие гипотезы.
- Продакт-менеджеры, которым нужен кликабельный/запускаемый прототип.
- Агентства и студии, автоматизирующие типовые MVP.

## MVP Scope (обязательные функции)

1. Регистрация и вход (JWT).
2. Создание проекта: название + описание идеи.
3. Конвейер агента из 6 фаз: Vision → Roadmap → Architecture → Structure → Code → Verify.
4. Хранение артефактов каждой фазы (документы + файлы кода) и их просмотр в UI.
5. Статус выполнения конвейера в реальном времени (polling).
6. Скачивание готового проекта одним ZIP-архивом.
7. Генерация работает офлайн (детерминированный template-движок) и опционально через
   LLM-провайдера (OpenAI-совместимый API), если задан ключ.
8. Docker Compose, автотесты, Swagger, документация.

## Future Scope (за месяц и далее)

- Диалоговое уточнение требований перед генерацией.
- Очередь задач (Redis + worker) и параллельные генерации.
- Push в GitHub-репозиторий пользователя, CI/CD-шаблоны.
- Автозапуск сгенерированного проекта в песочнице и авто-QA его API.
- Биллинг, тарифы, командные аккаунты.
- Больше стеков генерации (Spring Boot, Node/Nest, Next.js).

## Риски

| Риск | Митигация в MVP |
|---|---|
| LLM недоступен / нет ключа | Детерминированный template-движок по умолчанию |
| Сгенерированный код не запускается | Фаза Verify: синтаксическая проверка всех файлов |
| Долгая генерация блокирует API | Фоновое выполнение конвейера, статус через polling |
| Утечка чужих проектов | Все ресурсы привязаны к owner_id, проверка на каждом запросе |

## Что реально за неделю / за месяц

- **Неделя (этот MVP):** пп. 1–8 из MVP Scope.
- **Месяц:** диалоговое уточнение, GitHub-интеграция, песочница запуска, 2–3 стека генерации.
